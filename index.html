Supabase 設定說明
這個系統使用 Supabase 即時資料庫。請依以下步驟設定（約 10 分鐘），或先體驗 Demo 模式。
步驟 1 — 建立 Supabase 專案
前往 supabase.com，用 GitHub 帳號免費註冊，建立一個新 Project（任意名稱，Region 選 Southeast Asia）。
步驟 2 — 建立兩個資料表
在 Supabase 左側選 SQL Editor，貼上以下 SQL 並執行：
-- 專案資料表 create table projects ( id bigint generated always as identity primary key, no integer, code text, name text, unit text, eng text, pm text, stage text default 'ra', sit integer default 0, uat integer default 0, "statusRaw" text, "updatedBy" text, "updatedAt" timestamptz default now() ); -- 修改歷史表 create table history ( id bigint generated always as identity primary key, "docId" text, "projectName" text, changes jsonb, type text, "user" text, ts timestamptz default now() ); -- 開啟 Realtime alter publication supabase_realtime add table projects; alter publication supabase_realtime add table history;
步驟 3 — 複製 API 金鑰
左側選 Project Settings → API，複製：
• Project URL（例：https://abcxyz.supabase.co）
• anon public 金鑰（長串 JWT）
步驟 4 — 填入這個 HTML
用記事本開啟此 HTML，找到 ⚙️ 填入你的 Supabase 設定，貼入 URL 和 Key 後儲存。
步驟 5 — 部署到 GitHub Pages（免費、無限次更新）
1. 前往 github.com 登入後，點右上角 + → New repository
2. 名稱隨意（例如 project-tracker），選 Public，按 Create repository
3. 點 uploading an existing file，把這個 HTML 改名成 index.html 後拖曳上傳，按 Commit changes
4. 進入 Settings → Pages，Branch 選 main，資料夾選 / (root)，按 Save
5. 等約 1 分鐘，網址會顯示為 https://你的帳號.github.io/project-tracker/
之後更新只要重新上傳 index.html 即可，沒有次數限制。
